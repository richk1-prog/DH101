Play Game

2. Design statement. I wanted to make a sort of horror choose your own adventure, with a kind of Cthulhu style cosmic horror, with multiple different bad endings and only one way to prevent calamity. I would say for the most part other than the very first choice you can make, all your choices matter, while some choices lead to similar endings, for the most part each choice leads to a different way for it to end. I purposely chose to make this in python because I wanted to add content if someone didn't want to follow the rules, and because for me it's a resource I know how to use in this way. I do recommend you replay it a few times and do at least one playthrough where you type a number option that wasn't provided at every turn. I wanted to make almost every choice matter in some way because when the premise is trying to go against an eldritch horror every choice would matter. I made the first choice not matter, as a sort of tutorial choice, so the player could see you can make any choice, and get different results. I think the rules of Python and the 'input' function worked better for my idea because it allowed me to write code where I was prepared for anything and any output by the player with if, elif, and else statements to produce those outcomes.

4. Artist Statement
From this make I learned a lot about how important and how much effort it takes to not just create a game, but to create a game that accounts for even the little details, because when I was writing the code I had to jump around and make sure I had prepared eventualities for any and all outcomes a player might choose. However when it came to AI collaboration I hardly used it beyond having it make the different if, elif, and else statements, I never had it write any of the actual story, or any print or input statements. Which was more of a personal preference, it might have had something to do with the fact that I love video games so I didn't think it would be right for me to use AI much at all during this make. It also could just be because I knew what I was doing in writing the input and if, elif, and else statements in python, so I didn't see a need for AI collaboration in making the game. If I had to do things differently, I probably would have either expanded the game to more content, and also separate choices into multiple definitions so the code chunks weren't so long and hard to understand to anyone else.
